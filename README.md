Tools: Salesforce Classic & Lightning, Custom Objects, Record Types, Page Layouts, Validation Rules, Workflow/Process Builder, Approval Processes, Permission Sets, Email Templates

Project Overview:

Developed an end-to-end Student Admission Management System to automate and streamline enrollment, fee calculations, approvals, and communication workflows.

Key Achievements & Responsibilities:

Designed the data model using standard objects (Account, Contact) and custom Student__c to capture application & educational details.

Configured all Student__c fields, including:

Qualification, Stream, Percentage, University, Year, Achievements

Status, Fees

Auto-Increment Application ID

Built multiple Page Layouts, Field-Level Security, and Permission Sets for Student Users & Approver.

Implemented dynamic and conditional field behavior:

Auto-set Qualification based on inputs.

Conditional picklist values for Year depending on Graduation/Post Graduation.

Developed automation using Workflow/Process Builder:

Fees automatically assigned based on Stream:

IT / Computer → 50,000

Mechanical / Civil → 55,000

ENTC / Electrical → 60,000

Added Account & Contact related lists to Student__c for complete student profiling.

Created three users:

2 × Student Users

1 × Approval User

Designed a multi-step Approval Process:

Students submit application → Approver reviews

Status updates to Approved or Rejected

Configured Email Alerts with templates:

Auto-email for approved applications

Auto-email for rejected applications

Ensured compliance with admission criteria, data integrity, and accurate validation.
